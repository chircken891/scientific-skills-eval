```

</read_first>

<action>
检测skill间的互斥关系：

1. 功能完全重叠互斥：
   - 同名Function1的pair（如两个literature-review）
   - Coverage_Stages完全相同的pair
   
2. 依赖冲突互斥：
   - A依赖B但B不兼容A
   - MCP冲突风险分数≥4的pair
   
3. 顺序依赖：
   - A必须在B之前安装（如基础工具→专业工具）
   - B是A的上游依赖

检测维度：
- TSV的Integration_冲突风险(1-5)
- Coverage_Stages重叠度
- Function1名称匹配度
```

<acceptance_criteria>
- 识别出明确的互斥pair（≥1对）
- 识别出顺序依赖关系（≥1对）
- 冲突原因具体明确
- 输出写入03-CONFLICTS.md
