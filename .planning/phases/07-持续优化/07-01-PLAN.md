    The script measures 3 metrics for each of the 7 core skills:

    1. **Metric definitions**:
       - `RESPONSE_LATENCY_MS`: Time from request to first meaningful response (use `time` command or `date +%s%N` timestamps). Target: < 30000ms (30s).
       - `ORCHESTRATION_DELAY_MS`: Time scientific-do takes to route and dispatch a skill. Target: < 5000ms (5s).
       - `TOKEN_CONSUMPTION`: Estimated tokens per invocation (from response length approximation). Target: < 50000 tokens.

    2. **Timing function**: Create a `bench_skill()` function that:
       ```bash
       bench_skill() {
         local name="$1"
         local query="$2"
         local skill_dir="$3"

         echo "=== Benchmarking: $name ==="

         # Measure skill load time (SKILL.md read + parse)
         local start=$(date +%s%N)
         head -5 "$skill_dir/SKILL.md" > /dev/null
         local load_end=$(date +%s%N)
         echo "  Parse time: $(( (load_end - start) / 1000000 ))ms"

         # Simulate orchestration time (grep + match)
         start=$(date +%s%N)
         grep -q "triggers\|description" "$skill_dir/SKILL.md" 2>/dev/null
         local match_end=$(date +%s%N)
         echo "  Route time: $(( (match_end - start) / 1000000 ))ms"
       }
       ```

    3. **Benchmark all 7 core skills** with these representative queries:
       - deepxiv_sdk: "search for papers on CRISPR"
       - academic-paper-analysis: "analyze this paper methodology"

       - Paper-Polish-Workflow-skill: "polish manuscript for submission"
       - medsci-skills: "check PRISMA compliance"

    4. **Output format**: Produce a TSV table at the end:
       ```

       deepxiv_sdk	12	3	1093
       ...
       ```

    5. **Save output**: Write the TSV table to `.planning/phases/07-持续优化/benchmark-results.tsv` (append timestamp to filename if run multiple times).
