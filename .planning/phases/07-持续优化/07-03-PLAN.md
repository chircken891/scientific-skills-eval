|----------|-------------|
| User input -> Intent parsing (D-04) | Unparsed user query enters scientific-do's intent detection — potential injection |
| Feedback input -> feedback-state.json (D-15) | User's 1-5 rating and comment written to JSON file — tampering risk |
| Extension skill activation (D-19) | Extension skills activated on-demand from skills-extensions/ — privilege elevation risk |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-02 | Input Validation | Intent parsing step 1 | mitigate | Parse structured fields (keywords, scenarios) from frontmatter — do NOT eval user input. ECC pre-classifier as first gate |
| T-7-01 | Tampering | Feedback rating JSON write | mitigate | Sanitize comment: `jq --arg c "$COMMENT"` encodes as JSON string, preventing injection. Strip newlines before write |
| T-7-04 | Elevation of Privilege | Extension activation D-19 | mitigate | Only activate skills pre-downloaded to `~/.claude/skills-extensions/` and listed in config.json. No arbitrary URL activation |
</threat_model>
