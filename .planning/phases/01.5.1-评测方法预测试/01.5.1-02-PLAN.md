## Key Rules for Scoring

1. **D-03:** Score ONLY 1-2 best functions per skill. Do NOT score irrelevant dimensions.
2. **D-04:** Coverage is descriptive only. Do NOT assign numeric scores to coverage.
3. **D-05:** Security runs independently — record 4 checks (数据安全, 权限范围, 网络请求, 依赖来源) as PASS/FAIL separately from scores.
4. **D-06:** The scoring must NOT penalize focused, specialized skills. If a skill is excellent at one thing, it gets a high DepthScore even if it does nothing else.
5. **D-07/D-08/D-09:** <3.0=EXCLUDE, 3.0-4.0=CANDIDATE, >4.0=AUTO-RECOMMEND. Use 0.5 increments.
6. **D-10/D-13:** Any EXCLUDE classification MUST include a structured exclusion reason.

## Rubric Reference

The rubric document (01.5.1-SCORING-RUBRIC.md) defines 1-5 criteria for each function type. When scoring:
- For each sample, select 1-2 functions following the Function Selection Priority Rule
- Assign a 1-5 score per function using the rubric criteria
- DepthScore = average of selected function scores, rounded to nearest 0.5
- Map DepthScore to Threshold per D-07/D-08/D-09

**Sub-step C: Professional Depth Scoring (1-5 per function)**

3. Assign score 1.0-5.0 in 0.5 increments
4. Document specific evidence from the repo justifying the score

Score aggregation:
- If 2 functions selected: DepthScore = (Function1_Score + Function2_Score) / 2, rounded to nearest 0.5
- If 1 function selected: DepthScore = that function's score

**Sub-step D: Threshold Classification**
Apply D-07/D-08/D-09:
- DepthScore < 3.0 → Threshold = EXCLUDE
- DepthScore 3.0-4.0 (inclusive of both) → Threshold = CANDIDATE
- DepthScore > 4.0 → Threshold = AUTO-RECOMMEND

If Threshold = EXCLUDE, write exclusion reason in format: "[Check Failed] | [Evidence Quote] | [Why Matters]"

**Sub-step E: Integration Scoring (Retained)**
Score the same 5 integration dimensions 1-5 each:
1. 集成难度: How complex is installation (1=very hard, 5=one-command install)

3. 冲突风险: Overlap with other skills (1=high overlap, 5=unique)
4. 维护成本: Update frequency (1=abandoned, 3=sporadic, 5=active)
5. 上下文依赖: Context requirements (1=high dependency, 5=self-contained)

Integration_小计 = sum of 5 scores (range: 5-25)

- Coverage_Stages: Which of the 4 stages does this skill cover? (文献/分析/写作/投稿). Use comma-separation for multi-stage. Example: "文献,分析,写作"
