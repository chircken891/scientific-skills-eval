**Purpose:** Compare old scoring system (40/30/30 weighted sum, 100pt scale) with new system (1-5 professional depth rating + descriptive Tier 2) across all 10 samples

---

## Section 1: Overview

The old scoring system used an 8-dimension functional score (40%), 5-dimension integration score (30%), and 4-dimension coverage score (30%), totaling 100 points. This model penalized specialized skills because dimensions irrelevant to a skill's focus dragged down the overall score. For example, a citation management tool was scored on its "医学适配度" and "多模态" even though those dimensions are irrelevant to its stated purpose.

The new scoring system evaluates only 1-2 best functions per skill on a 1-5 professional depth scale, with coverage and integration becoming descriptive inputs for combination decisions rather than scored factors. This delta analysis compares the two systems across all 10 samples to validate D-06: that specialized skills are not penalized under the new methodology.

### Rationale for Zero EXCLUDE Items

The absence of EXCLUDE items in this pre-test sample is expected and does not indicate a rubric problem:

1. **Sample selection bias:** All 10 repositories were pre-curated as representative quality samples (9 GitHub-sourced repos with significant stars + 1 user-provided skill). None were intentionally selected to represent low-quality or broken skills.

2. **Floor effect:** The old evaluation already filtered repos by basic functionality — all scored 48+/100 under the old system. The new system, which only scores best functions, naturally produces higher scores for these already-functional repos.

3. **Natural distribution expectation:** In Phase 2 full evaluation (21 repos), the EXCLUDE category will likely be populated by genuinely non-functional or trivially-capable skills. The pre-test sample is too small and too curated to expose this category.

4. **Recommendation for Phase 2:** When evaluating the remaining 11-12 repos, ensure at least 2-3 are selected specifically to test the EXCLUDE threshold boundary (e.g., repos with minimal README-only documentation, single-prompt skills, or clearly abandoned repos). This will validate that the <3.0 threshold produces actionable exclusions.

## Section 5: Key Observations

### 1. Which repos changed most between old and new systems, and why?

**Biggest gainers:**
- paper-plot-skills: +5 positions (8th → 3rd). Old system penalized for lack of medical adaptation, multi-modal, and general writing capability. New system scores only Figure Generation (4.0/5) — a fair reflection of its genuine quality at publication figure generation.
- nature-skills: +2 positions (5th → 3rd). Journal-specific expertise (Nature-family journals) was penalized under old system for not covering broad domains. New system recognizes 4.0/5 for Figure Generation + 4.0/5 for Journal-Specific Skills.

**Biggest losers:**
- ARIS: -4 positions (4th → 8th). Old system valued its comprehensive 62-skill pipeline across 8 dimensions. New system scores only Literature Search (3.0/5) + Workflow Orchestration (3.5/5). The 3.0 Literature Search reflects that ARIS is not a dedicated search tool — it has basic search capability within its pipeline but lacks systematic review features. This is a more accurate assessment.
- academic-writing-skills: -2 positions (6th → 8th). Old system gave credit for LaTeX support, document parsing, and multi-skill writing suite. New system scores only Academic Writing (3.5/5) — between basic draft (3/5) and journal-adaptive with revision (4/5). Lacks journal templates and review simulation.

**Key insight:** The biggest rank changes occurred for repos where the old system gave credit for breadth features that are not directly related to the skill's stated purpose. The new system more accurately reflects domain-specific quality.

### 2. Were there any surprising scores that suggest rubric issues?

**Observation: ARIS Literature Search at 3.0/5 seems low but is consistent with the rubric.**
The rubric defines 3/5 as "1-2 databases, keyword search only, no systematic query tools, manual filtering." ARIS has literature search as a supporting capability within its research pipeline — it integrates with existing search capabilities but does not provide dedicated systematic review tools (PICO formulation, Boolean operators, quality filtering). The 3.0 score is defensible. However, if ARIS's literature search is more capable than described, future Phase 2 evaluation should verify against actual implementation.

**Observation: academic-writing-skills at 3.5/5 is conservative.**
The rubric defines 3/5 as "basic structure, single template, limited revision" and 4/5 as "3+ journal templates, revision support, review checklist." academic-writing-skills has paper-audit (multi-reviewer system) which goes beyond basic revision. However, without confirmed journal-specific templates, 3.5/5 is a reasonable conservative estimate. Phase 2 evaluation should examine the actual LaTeX templates to determine if 4.0/5 is more appropriate.

**Observation: All 6 AUTO-RECOMMEND repos scored exactly 4.0 or 4.5.**
No repo achieved 5.0 in its selected function(s). This could indicate the rubric is calibrated conservatively, or that none of the 10 samples truly achieve "best-in-class" status in their selected domains. The 5/5 criteria are deliberately demanding (e.g., "5/5 Figure Generation requires journal-specific typography, style transfer from papers, multi-panel composition, colorblind-safe palettes"). For Phase 2, if many repos score 4.0 without achieving 5.0, the 4/5 criteria may need to be reconsidered for grading on a curve against the actual population.

### 3. Does the pipeline scoring approach work for academic-pipeline? Any issues?

academic-pipeline scored 4.0/5 as Workflow Orchestration — solid performance for a 10-stage pipeline with comprehensive stage coverage.

**Issues found:**
- **Documentation scarcity:** academic-pipeline is user-provided and lacks the extensive documentation of other repos. Scoring required extrapolation from the "10-stage serial pipeline" description. In Phase 2, if a skill's documentation is insufficient for scoring, it should be flagged for manual review before evaluation rather than scored conservatively.
- **Pipeline vs. skill boundary:** academic-pipeline scored 4.0 alongside paper-plot-skills (single specialized tool) and nature-skills (6-skill suite). The AUTO-RECOMMEND classification is appropriate for a well-designed pipeline, but the comparison is somewhat apples-to-oranges — the 4.0 for academic-pipeline means something different from the 4.0 for a figure generation tool. The rubric handles this correctly by scoring each against its own category's criteria.

### 4. Are integration scores consistent with expectations?

Integration scores were mostly carried over from the old evaluation with minor adjustments. Key observations:

- scientific-agent-skills has the highest integration score (20/25), consistent with its status as the largest and best-documented skill library.
- AI-Powered-Literature-Review-Skills has the lowest integration score (15/25), consistent with its Docker-dependency complexity and overlap with ARIS.
- academic-pipeline scored 15/25 — conservative due to unknown maintenance status and MCP compatibility.

The integration scores are consistent and can serve as combination input for Phase 2.

### 5. Coverage descriptions: do they provide useful combination guidance?

The coverage descriptions use the 4-stage model (文献/分析/写作/投稿) and provide domain-specific context:

- medsci-skills and academic-pipeline cover all 4 stages — useful for users wanting end-to-end coverage
- paper-plot-skills and citation-assistant cover only 1 stage each — useful for users wanting to supplement other skills
- scientific-agent-skills and AI-Research-SKILLs cover 3 stages — useful for broad-coverage use cases

These descriptions directly enable the Tier 2 combination decisions that the new system requires.

## Section 7: Recommendations for Phase 2

1. **Rubric verification for borderline cases:** Before Phase 2 scoring, verify ARIS literature search implementation and academic-writing-skills template support. These two repos have the most uncertainty in the current scores.

2. **EXCLUDE category testing:** Select 2-3 deliberately low-capability repos for Phase 2 evaluation (e.g., repos with minimal README, single-prompt capability, or no recent maintenance). This validates the <3.0 threshold produces actionable exclusions.

3. **5/5 criteria calibration:** If Phase 2 reveals many repos scoring 4.0 but none reaching 5.0, the 4/5 criteria may need to be split into 4.0 and 4.5/5.0 tiers to provide more differentiation within the AUTO-RECOMMEND category.

4. **Pipeline scoring guidance:** For future pipeline/orchestrator repos, require minimum documentation: stage count, checkpoint locations, error handling approach, and data flow schema. Without these, conservatively score at 3.5/5.

5. **Security veto workflow testing:** Include at least 1 repo with known security concerns in Phase 2 to validate the exclusion workflow (D-10/D-13: veto + structured reason + human review).

6. **Integration score role in Phase 2:** Integration scores (retained from old evaluation) should serve as the primary differentiator within AUTO-RECOMMEND tier. Higher integration scores (e.g., scientific-agent-skills at 20/25) indicate better combinability. Recommend using integration score as secondary sort key after DepthScore.

7. **Coverage descriptions for combination:** The Tier 2 coverage descriptions should be used for manual combination decisions (D-12: human confirms final portfolio). For example, paper-plot-skills (分析) + medsci-skills (全流程) + citation-assistant (文献) would create a complementary combination covering all 4 stages.
