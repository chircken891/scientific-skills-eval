# 排除理由模板

**依据：** D-10（排除项必须附理由） + D-13（自动生成，人工可复核）

## 模板结构

每条排除理由包含三个字段，用管道符分隔以便在TSV中存储：

```
[检查项名称] | [证据引用] | [为何重要]
```

## 字段说明

### 1. 检查项名称

被触发的评估维度。必须是以下之一：
- FunctionDepthScore (< 3.0)
- SecurityVeto (任一安全检查FAIL)

### 2. 证据引用

来自skill仓库的具体证据。格式：
- 对于FunctionDepthScore: "该skill的[X功能]仅具备[具体限制说明]，不符合[对应分值]标准"
- 对于SecurityVeto: "[检查项名称]: [具体文件路径]: [具体问题描述]"

### 3. 为何重要

对医学研究场景的具体影响。格式：
- "科研场景需要[X能力]，该skill无法满足因为[具体原因]"

## 完整示例

### 示例1: 功能深度不足

```
FunctionDepthScore | 该skill的Literature Search功能仅支持单数据库关键词搜索，无系统综述方法论、无质量过滤、无去重能力，不符合3/5标准 | 流行病学研究需要进行系统综述（PRISMA标准），单数据库无过滤搜索会产生大量不相关结果，无法支持方法学严谨的文献检索
```

### 示例2: 安全否决

```
SecurityVeto-数据安全 | src/auth.ts:15: 将API密钥明文存储到~/.credentials.json | API密钥泄露可能导致第三方数据平台被盗用，违反科研数据安全规范
```

## 在TSV中的使用

排除理由写入SCORING-TEMPLATE.tsv的`ExclusionReason`列时，将"[字段1] | [字段2] | [字段3]"放入一个单元格。用管道符分隔三个字段。
