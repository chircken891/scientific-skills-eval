<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Shell args -> JSON serialization | User-provided text flows through shell positional args into JSON |
| mkdir lock -> concurrent write protection | Multiple concurrent invocations share feedback-state.json |
| GSD_PROJECT_ROOT -> output file path | Root path determines SD-*.md write location |
| archives -> long-term storage | Archived entries accumulate unbounded |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | JSON injection via intent text | mitigate | All fields serialized via node -e JSON.stringify(); shell never concatenates JSON |
| T-09-02 | Denial of Service | Orphan .lock dir blocks writes | mitigate | Stale lock guard: if lock dir exists and mtime > 10s, remove and retry |
| T-09-03 | Information Disclosure | SD-*.md outside allowed path | mitigate | Validate output path starts with GSD_PROJECT_ROOT/.planning/phases/ |
| T-09-04 | Tampering | Archive corruption from concurrent writes | accept | Archive writes protected by shared mkdir lock |
</threat_model>
