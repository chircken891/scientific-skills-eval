## 讨论的问题

### 1. 测试执行方式
**问题:** 如何执行测试？
**选项:**
- A. 手动调用 — Claude Code session中逐个调用skill测试
- B. 自动化脚本 — 写脚本批量调用，自动化验证
- C. 混合模式 — 手动测试为主，部分自动化辅助

**选择:** A. 手动调用
**原因:** skill在Claude Code环境中，手动调用最真实

---

### 2. 失败处理
**问题:** 测试失败后怎么处理？
**选项:**
- A. 立即修复 — 测试失败后立即尝试修复
- B. 记录问题 — 只记录问题，不当场修复
- C. 分级处理 — 关键功能失败→立即修复，边缘情况→记录

**选择:** C. 分级处理
**原因:** 平衡测试效率和灵活性

---

### 3. Scientific-Do测试
**问题:** 如何测试Scientific-Do协调器？
**选项:**
- A. 预设场景测试 — 用固定测试输入验证路由正确性
- B. 自由测试 — 随意描述任务，看协调器如何响应
- C. 两者结合 — 预设场景验证核心 + 自由测试边缘情况

**选择:** C. 两者结合
**原因:** 兼顾核心功能和边缘情况

---

### 4. 测试环境
**问题:** 在哪里执行测试？
**选项:**
- A. 当前session — 在当前Claude Code session中测试
- B. 新建session — 启动全新的Claude Code session测试
- C. 分开测试 — 每个skill开一个独立session测试

**选择:** B. 新建session
**原因:** 确保测试环境干净，不受之前操作影响
