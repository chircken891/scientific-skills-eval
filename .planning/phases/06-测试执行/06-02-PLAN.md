<coordinator_components>
## Scientific-Do Coordinator Architecture

### 3-Tier Routing (skill-router.ts)
1. **Exact Match**: Full stage match -> direct skill
2. **Fuzzy Fallback**: Keyword partial match -> candidate skills
3. **Intelligent Tuning**: No match -> default to deepxiv_sdk

### Intent Parser (intent-parser.ts)
- Extracts: stage, taskType[], domainKeywords[], confidence
- Input: Natural language request
- Output: Structured intent object

### Dependency Chain (skill-router.ts)
- buildDependencyChain(tasks[]): string[]
- Returns ordered skill sequence for task chain

### HARD-GATE Nodes
| Stage | Gate Condition | Expected Behavior |
|-------|---------------|-------------------|
| Before literature search | None | Allow |
| Before planning | Missing research | Pause with prompt |
| Before writing | Missing study design | Pause with prompt |
| Before execution | User not confirmed | Wait for confirmation |
</coordinator_components>
