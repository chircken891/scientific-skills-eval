## New Scoring System (Two-Tier Architecture)

### Tier 1: Professional Depth Scoring (DECISION DRIVER)

**Step 1: Function Selection** (Priority Rule)
1. If repo README/description declares a specialization → score that first
2. If multiple specializations declared → score the one with strongest evidence
3. If no clear specialization → score the 2 functions with most concrete implementation detail

**Step 2: Score each selected function (1-5, 0.5 increments)**
- Use 01.5.1-SCORING-RUBRIC.md for detailed criteria per function type
- Document specific evidence from repo for each score

**Step 3: Calculate DepthScore**
- 1 function: DepthScore = that function's score
- 2 functions: DepthScore = average, rounded to nearest 0.5

**Step 4: Apply Threshold**
- < 3.0 → EXCLUDE (requires structured exclusion reason per D-10)
- 3.0 - 4.0 → CANDIDATE
- > 4.0 → AUTO-RECOMMEND

### Tier 2: Supplementary Info (NOT scored, for combination decisions)

**Integration Scores (5 dimensions, 1-5 each):**
- 集成难度, MCP兼容, 冲突风险, 维护成本, 上下文依赖
- Total: 5-25

**Coverage Description (DESCRIPTIVE, not scored):**
- Coverage_Stages: literature/analysis/writing/submission
- Coverage_Description: 2-3 sentences

### Security (Independent Veto — unchanged)
- 4 checks: 数据安全, 权限范围, 网络请求, 依赖来源
- Any FAIL → VETOED + structured reason
