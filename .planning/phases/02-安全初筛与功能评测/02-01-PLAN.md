# Security Check Criteria (D-05)

| Check | Pass | Fail |
|-------|------|------|
| 数据安全 | No access to PII/medical records/credentials | Accesses/stores sensitive data |
| 权限范围 | Requests minimal necessary permissions | Requests excessive permissions |
| 网络请求 | No suspicious external communication | Sends data to unknown endpoints |
| 依赖来源 | Uses trusted package registries | Uses untrusted/unknown dependencies |

# Scoring Dimensions (D-07, D-08, D-09, D-10)

**Functional (40%):** 核心能力, 医学适配度, 长文本适配, 统计/数学能力, 代码生成, 工具调用效率, 自主兜底, 多模态 — each 1-5pt

**Integration (30%):** Claude Code集成难度, MCP兼容性, 冲突风险, 维护成本, 上下文依赖 — each 1-5pt

**Coverage (30%):** 领域覆盖, 科研阶段覆盖, 互补性, 冗余度 — each 1-5pt

# Scorecard Output Format (D-12)

Each repo output must include:
- Security: 4 items with PASS/FAIL + reason if FAIL
- Functional: 8 dimension scores (1-5 each) + subtotal
- Integration: 5 dimension scores (1-5 each) + subtotal
- Coverage: 4 dimension scores (1-5 each) + subtotal
- Weighted total: Functional×40% + Integration×30% + Coverage×30%

For each of the 9 sampled repositories, perform the following evaluation:

**Step 1: Clone/Fetch Repository**
Use `git clone --depth 1 <repo_url>` to get the latest code for evaluation.

**Step 2: Security Check (D-05)**
Examine files for:
- 数据安全: Look for credential storage, API key handling, data export features, PII access patterns
- 权限范围: Check package.json, manifest, permissions requested
- 网络请求: Review API calls, fetch requests, external service integrations
- 依赖来源: Check package.json, requirements.txt, go.mod for dependency origins

For each check, record PASS or FAIL with specific evidence (file:line references).

**Step 3: Score Evaluation (D-07 to D-10)**
Score each dimension 1-5:
- 1 = Very poor / almost none
- 2 = Basic / limited
- 3 = Adequate / some capability
- 4 = Good / solid capability
- 5 = Excellent / best-in-class

**Functional Dimensions (8×5=40pts max):**
1. 核心能力: Main function quality for stated purpose
2. 医学适配度: Fit for epidemiology/biostatistics/oncology workflows
3. 长文本适配: Ability to handle long papers (MiniMax 4M token context)
4. 统计/数学能力: Statistical formulas, mathematical reasoning
5. 代码生成: R/Python script generation quality
6. 工具调用效率: Tool invocation count and appropriateness
7. 自主兜底: Self-correction, error handling
8. 多模态: Chart/image understanding

**Integration Dimensions (5×5=30pts max):**
1. Claude Code集成难度: Installation complexity, config overhead
2. MCP兼容性: MCP protocol support (yes=5, partial=3, no=1)
3. 冲突风险: Overlap/conflict with other skills (lower overlap = higher score)
4. 维护成本: Update frequency, community support (active=5, abandoned=1)
5. 上下文依赖: Does it require long context (MiniMax makes this less critical)

**Coverage Dimensions (4×5=30pts max):**
1. 领域覆盖: Coverage of epidemiology/biostatistics/oncology
2. 科研阶段覆盖: Which stage: literature→analysis→writing→submission
3. 互补性: Works well with other skills
4. 冗余度: Duplication with other skills (less duplication = higher score)
