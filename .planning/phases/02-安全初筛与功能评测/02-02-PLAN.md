## New Scoring System (Same as Plan 02-01)

**Tier 1: Professional Depth Scoring**
1. Function selection (1-2 functions per priority rule)
2. Score each function 1.0-5.0 in 0.5 increments using SCORING-RUBRIC.md
3. DepthScore = average of selected function scores
4. Threshold: <3.0 EXCLUDE / 3.0-4.0 CANDIDATE / >4.0 AUTO-RECOMMEND

**Tier 2: Supplementary Info (NOT scored for threshold)**
5. Integration scores (5 dimensions, 1-5 each)
6. Coverage description (stages + 2-3 sentences)

**Security (Independent Veto)**
7. 4 checks: PASS/FAIL → APPROVED or VETOED

## Key Reminders from Phase 01.5.1

- For pipeline/orchestrator repos: score Workflow Orchestration, not individual stage capabilities
- For broad libraries (135+ skills): score only the 1-2 strongest functions
- Use SCORING-RUBRIC.md function type tables as the authoritative scoring criteria
- EXCLUDE repos MUST have structured reason: [check name] | [evidence] | [why matters]
- If a repo scores < 3.0 → EXCLUDE + structured reason
- If a repo has ANY security FAIL → VETOED + structured reason
